# Security Checklist — Рубеж ИИ

Чеклист проверяется на каждой итерации, итоговая проверка — в итерации 16.
`[ ]` — не выполнено, `[x]` — выполнено и проверено тестом/ревью.

## Секреты и логирование

- [ ] Raw secrets никогда не пишутся в application logs
- [ ] Есть автотест на отсутствие утечки секретов в логах
- [ ] Структурное логирование без чувствительных полей (slog / structlog)
- [ ] Секреты конфигурации читаются из env / docker secrets, не из кода
- [ ] `.env` в `.gitignore`; в репозитории только `.env.example`

## Персональные данные и обезличивание

- [ ] Внешние LLM по умолчанию получают только masked text
- [ ] Raw corporate data в `external` / `russian_cloud` запрещён политикой
- [ ] Псевдонимизация обратима только через зашифрованный mapping
- [ ] `pseudonym_mappings` — отдельная таблица с шифрованием значений (AES-GCM)
- [ ] Risk scoring покрывает ПДн, секреты, коммерческие данные

## Аудит

- [ ] `audit_events` — append-only (триггер БД запрещает UPDATE/DELETE)
- [ ] Каждое решение policy engine журналируется
- [ ] Audit хранит риск-классы и masked representation, не raw
- [ ] Audit replay восстанавливает цепочку: запрос → сущности → маскирование → решение → модель → ответ → проверка

## LLM и prompt injection

- [ ] Базовая проверка prompt injection во вводе
- [ ] Пост-проверка ответа модели на утёкшие секреты
- [ ] Режимы доверия моделей учитываются policy engine
- [ ] Mock-провайдер не выполняет реальных внешних вызовов

## Авторизация и доступ

- [ ] Auth-middleware проверяет роль на каждом защищённом запросе
- [ ] RAG-поиск фильтруется по ACL пользователя
- [ ] Роли разграничивают доступ к политикам, аудиту, инцидентам, моделям

## Инфраструктура

- [ ] Внутренние сервисы недоступны снаружи docker-сети
- [ ] Healthcheck'и настроены для всех долгоживущих сервисов
- [ ] Лимиты размера загрузки документа и длины запроса
- [ ] Образы Docker зафиксированы по конкретным тегам
- [ ] Версии образов периодически ревизуются с учётом накопленных CVE

## Зависимости и качество

- [ ] Все зависимости зафиксированы в lock-файлах (`package-lock.json`, `go.sum`, `uv.lock`)
- [ ] Нет экспериментальных библиотек без обоснования
- [ ] Нет файлов > 500 строк, функций > 60 строк без обоснования

## Тестирование безопасности

- [ ] Unit-тесты на каждый regex-детектор
- [ ] Тест отсутствия утечки секретов в логах
- [ ] Тест append-only поведения `audit_events`
- [ ] Тест ACL-фильтрации RAG
- [ ] e2e smoke-тест главного MVP-сценария
