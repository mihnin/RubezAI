# Экран: Documents (`/documents`, `/documents/:id`)

UX-spec. Реализуется на основе API из Итерации 10 (`/api/documents`).
Worker парсит PDF/DOCX, делает chunking, кладёт chunks с embeddings в
pgvector.

## Структура

### Drop-zone

- Высота 160, dashed border (`border-strong`, opacity 0.5), radius `md`,
  text-secondary.
- При drag-over — bg `info-bg`, border-color `accent`, +1px solid.
- Принимает только `.pdf`, `.docx`. Mime-проверка на клиенте и сервере.
- Лимит 50 МБ (см. THREAT_MODEL T9).

### Таблица документов

- Колонки: `name` (text-base + иконка типа), `size`, `uploaded_at`
  (relative + tooltip UTC), `status` (chip), действия (3-точечное
  меню).
- Sort: по умолчанию по `uploaded_at desc`.
- Pagination 50 per page.

### Статусы (chip)

| Статус | Color | Иконка |
|--------|-------|--------|
| `queued` | `text-muted` | `clock` |
| `parsing` / `chunking` / `embedding` | `info` | `loader` (вращ.) |
| `ready` | `success` | `check-circle` |
| `failed` | `danger` | `alert-circle` |
| `unsupported` | `warning` | `x-circle` |

В колонке статуса для активных стадий — chip + микро-progress-bar
(2px, ширина по реальному прогрессу `processed/total`).

### Detail-страница

- Bento-grid: «Метаданные» (большая карточка) + «Статус» (узкая).
- Список chunks — collapsible. По умолчанию **маскированный** текст;
  toggle «показать с маскированием» (всегда так — raw не доступен
  в UI; чекбокс показывает/скрывает highlight псевдонимов).
- Action-bar: «Использовать в чате» (создаёт сессию с прикреплённым
  документом), «Скачать оригинал», «Удалить».

## Состояния

| State | Описание |
|-------|----------|
| Empty | Hero «Загрузите PDF/DOCX, чтобы добавить его в базу знаний» |
| Uploading | Прогресс-бар над таблицей: «Договор-12.pdf — 42%» |
| Upload error | Toast `danger`: «Не удалось загрузить: …» |
| Processing | В строке таблицы — chip + progress |
| Processing failed | Chip `failed` + tooltip с reason; action «Повторить» |
| Permission denied | Detail-страница вместо контента — banner: «У вас нет доступа к этому документу» |
| Delete confirmation | Dialog: «Удалить документ навсегда? Это удалит все чанки и embeddings.» Кнопка danger «Удалить» |

## Поведение

1. Drop → `POST /api/documents` (multipart). Возвращает `document_id`,
   статус `queued`.
2. Worker обрабатывает → polling каждые 2s **с exponential back-off**
   (m4 ревью этапа A): `interval = min(8000ms, 2000 * 2^retries)`,
   где `retries` инкрементируется при отсутствии изменений статуса.
   Сбрасывается при любом изменении. Защита от лавины на 100+
   документах. SSE-push — пост-MVP.
3. Detail-страница — `GET /api/documents/:id` + `GET
   /api/documents/:id/chunks`.

## Безопасность

- Никакого raw-просмотра содержимого; всегда masked.
- Скачивание оригинала разрешено **только владельцу и admin** (см.
  ACL); audit-event `document_downloaded` обязателен.
- Удаление — soft-delete (флаг + ACL-блок); полное удаление — пост-MVP.

## Accessibility

- Drop-zone имеет видимый «Выбрать файл» как fallback (без drag-drop).
- Таблица навигируема стрелками; Enter — открывает detail.
- Прогресс-бары — `aria-valuenow / valuemin / valuemax`.

- Покрыто всё MVP, плюс расширения (`Использовать в чате` —
  заглушка для итерации 11 RAG).
- Не закрыт момент: polling vs SSE (вынесен в пост-MVP — обоснованно).
