# UX-спецификация — чат с документами и контролируемый вывод ПДн (Итерация J)

Fallback-режим: Stitch MCP в сессии недоступен → текстовая спека + wireframes
(см. docs/design/ui-system.md за токенами). Визуальные макеты — в Stitch после
перезапуска CC. Флоу/контракты — docs/design/chat-pii-flow.md.

## Дизайн-токены (из существующего кода)

- Фон: `bg-slate-950` (полотно), карточки `bg-slate-900/60`, бордеры
  `border-slate-800`. Акцент: `cyan-500` (CTA), `cyan-300/400` (ссылки/иконки).
- Семантика: external/опасность — `amber`/`red`; trusted_local/успех —
  `emerald`. Радиусы `rounded-xl`, тени `shadow-lg shadow-cyan-500/20`.
- Доступность: WCAG 2.2 AA, focus-ring `focus:ring-2 ring-cyan-500/20`,
  `aria-live` для статусов, `role="alert"` для ошибок.

## Экран 1 — Чат: picker (cloud/local) + attach

```
┌───────────────────────────────────────────────────────────────┐
│ Чат                                  [ ◇ deepseek-cloud · …▾ ] │  ← picker
│ Сессия a1b2c3d4                       ⚠ облачная — данные за    │
│                                          контур                 │
├───────────────────────────────────────────────────────────────┤
│  (лента сообщений)                                              │
│                                                                 │
├───────────────────────────────────────────────────────────────┤
│ [📎]  ┌─────────────────────────────────────────┐   [ ➤ ]      │
│       │ Введите сообщение…                       │             │
│       └─────────────────────────────────────────┘             │
│  📎 Договор_тест_ПДн.docx  ✓ обработан · 12 ПДн   [✕]          │  ← attach-чип
└───────────────────────────────────────────────────────────────┘
```

**Picker (dropdown)** — две группы с заголовками:

```
┌─ Провайдер ─────────────────────────┐
│ ОБЛАЧНЫЕ  ⚠ данные уходят за контур │
│   ◉ deepseek-cloud      external    │
│   ○ openai-gpt          external    │
│   ○ anthropic-claude    external    │
│ ЛОКАЛЬНЫЕ  🛡 в периметре, raw      │
│   ○ (нет включённых)                │
│─────────────────────────────────────│
│ Модель: [ deepseek-chat        ]    │
└─────────────────────────────────────┘
```

- `📎` attach → загрузка `POST /api/documents`, чип показывает статус
  (`обрабатывается…` → `✓ обработан · N ПДн` / `✗ ошибка`). Send заблокирован,
  пока документ не `done`.
- Бейдж в шапке: `⚠ облачная` (amber) для external, `🛡 локальная` (emerald)
  для trusted_local. Состояние «нет активных провайдеров» — как сейчас.

## Экран 2 — Гейт перед отправкой в облако (модал)

```
┌──────────────────────────────────────────────────────────┐
│ ⚠  Данные уйдут за контур предприятия                     │
│    Провайдер deepseek-cloud (облако). Перед отправкой —    │
│    проверьте обезличенный текст.                           │
│                                                            │
│  ОБЕЗЛИЧЕННЫЙ ТЕКСТ (уйдёт в LLM):                         │
│  ┌──────────────────────────────────────────────────────┐ │
│  │ Заказчик ФИО_001, паспорт ПАСПОРТ_001, карта          │ │
│  │ КАРТА_001, тел. ТЕЛЕФОН_001…                           │ │
│  └──────────────────────────────────────────────────────┘ │
│                                                            │
│  НАЙДЕНО И ЗАМЕНЕНО:  всего 12   риск: ● critical          │
│   PERSON 3 · PASSPORT 1 · BANK_CARD 1 · SNILS 1 · INN 2 …  │
│   классы: ● pii  ● secret                                  │
│                                                            │
│                       [ Отмена ]   [ ➤ Отправить в облако ]│
└──────────────────────────────────────────────────────────┘
```

- Источник данных — `POST /api/chat/preview` (возвращает `preview_token`,
  `sanitized_text`, `entities`, `risk`). Статистика считается из `entities`.
- «Отправить» → `POST /api/chat` с `preview_token` (тот же текст, что показан —
  гарантия J.0). Индикатор «Обрабатываем…» (экран 4) показывается ДО гейта,
  пока идёт preview.
- Для `trusted_local` модал **не показывается** (данные не покидают периметр).
- A11y: заголовок `role="alertdialog"`, фокус на «Отмена», Esc = отмена.

## Экран 3 — Ответ ассистента: псевдонимы + reveal

```
┌────────────────────────────────────────────────────────┐
│ 🤖  Договор с ФИО_001 на сумму СУММА_001 подписан.       │
│     Реквизиты карты КАРТА_001 приняты.                   │
│                                                          │
│     allow_masked · deepseek-cloud      [👁 Показать      │
│                                          реальные данные]│
└────────────────────────────────────────────────────────┘
            │ (клик)
            ▼
┌────────────────────────────────────────────────────────┐
│ 🤖  Договор с Соколовой Екатериной… на сумму 185 000 ₽   │
│     подписан. Реквизиты карты 2202 2012 3344 5566 …      │
│                                                          │
│     allow_masked · deepseek-cloud   🔓 раскрыто (вами,   │
│                                        14:05) · аудит    │
└────────────────────────────────────────────────────────┘
```

- По умолчанию — псевдонимы. Кнопка `👁 Показать реальные данные` →
  `POST /api/chat/messages/{id}/reveal` (детерминированно, `no-store`).
- После раскрытия — бейдж `🔓 раскрыто` (кто/когда), действие в аудите
  (`response_revealed`). Повторно скрыть — toggle (локально, без запроса).
- Если у псевдонима нет mapping (LLM перефразировал) — остаётся псевдонимом.

## Экран 4 — Индикатор обработки ПДн

```
   ⟳  Обрабатываем персональные данные…
      Детекторы + локальная LLM-проверка перед выходом за контур
```

- Тонкая полоса/спиннер `cyan`, `aria-live="polite"`. Показывается во время
  `preview` (и при attach-парсинге документа). Не блокирует ввод нового текста.

## Экран 5 — Документы: статистика + выгрузки

```
┌───────────────────────────────────────────────────────────────┐
│ 📄 Договор_тест_ПДн.docx          ✓ done · 03.06.2026          │
│    ● critical   12 ПДн: PERSON 3 · PASSPORT 1 · BANK_CARD 1 …  │
│    [⬇ Оригинал]  [⬇ Обезличенный (.txt)]  [🔎 Маппинги/история]│
└───────────────────────────────────────────────────────────────┘
```

- «Оригинал» — `GET /api/documents/{id}/download` (есть).
- «Обезличенный» — новый endpoint: `text/plain` склейка sanitized-чанков
  (метка «форматирование не сохранено»; masked-DOCX — техдолг).
- «Маппинги/история» — список сущностей по документу (тип → псевдоним →
  raw_hash, без raw) + ссылка на запись/номер документа.
- Скачивание обезличенной версии аудируется (`document_masked_downloaded`).

## Состояния (для всех экранов)

- **Пусто** / **загрузка** (skeleton) / **ошибка** (`role="alert"`).
- **deny/escalate** в чате — баннер блокировки (как сейчас), reveal недоступен.
- **leak warning** — если детектор утечки сработал, ре-маскирование (бэкенд),
  пользователю виден псевдоним.
- **reveal forbidden** (не владелец сессии) — кнопка скрыта/недоступна.
